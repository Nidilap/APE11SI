COM FLOW

Comando O que faz
git flow init Comando obrigatório para começar a usar o git flow no projeto. Ao executar esse comando, você precisará responder algumas perguntas relacionadas a branch e estrutura do projeto.
git flow feature start Cria uma nova feature (implementação). Isso faz com que seja criada uma nova branch para desenvolvimento.
git flow feature finish Finaliza o desenvolvimento da feature e mescla com a branch principal, deletando a branch da feature.
git flow feature publish Publica a funcionalidade para o repositório central
git flow feature pull Obtem uma funcionalidade do repositório central.
git flow release start Cria uma nova versão.
git flow release publish Publica uma nova versão para o repositório central.
git flow release finish Finaliza a nova versão.
git flow hotfix start Cria um novo hotfix (uma correção no sistema).
git flow hotfix finish Da um merge em develop e na master do hotfix.
